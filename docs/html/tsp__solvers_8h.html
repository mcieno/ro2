◆ _xopt2subtours()

void _xopt2subtours ( const instance *  problem,
const double *  xopt,
size_t *  next,
size_t *  comps,
size_t *  ncomps,
size_t(*)(size_t, size_t, const instance *)  pos 
)

Given a CPLEX-generated solution, create a more convenient representation.

Given the incumbent solution xopt, where xopt[e] = 1 <==> edge e was selected, populate provided arrays next and comps so that next[i] = j <==> the tour goes from node i to node j and comps[i] = k <==> node i is part of the k-th subtour.

The number of subtours is written to ncomps, hence xopt is a valid TSP solution iff ncomps == 1.

Parameters
problemPointer to the instance structure.
xoptCPLEX incumbent solution. xstar[xpos(i, j)] == 1 iff the edge was selected.
nextArray of adjacencies to be filled. next[i] = j means that there is an arc going from node i to node j.
compsArray of components indices to be filled. comps[i] = k means that node i belongs to connected component k.
ncompsPointer to an integer where to store the number of connected components in the solution. If 1, the solution is a tour.
posPointer to a function that given coordinates i and j returns the position in xopt fo x(i,j).

◆ LoopF_model()

void LoopF_model ( instance *  problem)

Solve with Dummy "Branch and Cut" model (variant 'F'), restarting cplex after every intermediate solution.

This model is similar to Loop_model(). The main difference is that it starts with a loose EPGAP and tightens it iteration after iteration, until a single component is found, possibly sub-optimal. At that point, the default MIP optimizer is run.

Parameters
problemPointer to the instance structure.

◆ LoopM_model()

void LoopM_model ( instance *  problem)

Solve with Dummy "Branch and Cut" model (variant 'M'), restarting cplex after every intermediate solution.

This model is similar to Loop_model(). The main difference is that it starts with a loose EPGAP and a small limit of solutions. It tightens the gap and increases the solution limit until a single component is found, possibly sub-optimal. At that point, the default MIP optimizer is run.

Parameters
problemPointer to the instance structure.

◆ LoopX_model()

void LoopX_model ( instance *  problem)

Solve with Dummy "Branch and Cut" model (variant 'X'), restarting cplex after every intermediate solution.

This model is similar to Loop_model(). The main difference is that it starts with a tight EPGAP and a large limit of solutions. It looses them according to the number of components it found at each solution, until a single component is found, possibly sub-optimal. At that point, the default MIP optimizer is run.
